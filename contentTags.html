Content Categories

We can loosely group the elements in different categories. There are three types of categories as below.

  • Main content categories - those share common rules among the elements
  • Form-related content categories
  • Specific content categories

Main content categories

Main content categories are sub divided into eight different categories which are discussed below.

  1. Metadata content

    Elements belonging to this category are <base>, <link>, <meta>, <noscript>, <script>, <style> and <title>.

  2. Flow content

    All of the elements of flow control category goes inside body element.

Paragraph : <p></p>

Paragraphs are used to represent a block of text separated from adjacent blocks.

Paragaphs are by default block-level elements.

<p>This is a paragraph</p>

Generally <hr< elements represent a thematic break between paragraph-level elements.

Emphasize : <em></em>

Emphasize element used to emphasize any text. It makes the text italic. It is a inline-level element. 

<p>I am <em>very</em> excited to learn html.</p>

I am very excited to learn html.

Strong : <strong></strong>

Emphasize element used to emphasize any text. It makes the text little bold. It is a inline-level element. 

<p>I am <strong>very</strong> excited to learn html.</p>

I am very excited to learn html.

Abbreviation Element : <abbr>

These elements are used to express the abbreviation. For the first time use of abbreviation we need to provide full expansion. Generally only using <abbr> alone cannot support the purpose. We need to provide 'title' attribute with <abbr> to express the full expansion as a tooltip. The abbreviated text by default rendered with dooted underline.

List specific elements

List Items element : <li>

This element represents an item in a list. This element must be wrapped with parent element: an ordered list, an unordered list or a menu.

We will see the usage of <li> in these three elements. It can accept two attributes named value and type.

There are two types of lists.

  1. Ordered list
  2. Unordered list

Ordered List : <ol>

We need ordered list when we like to make the list of the items in order (normal or reverse)

Below is an example.


        <ol>
            <li>Mix flour, baking powder, sugar, and salt.</li>
            <li>In another bowl, mix eggs, milk, and oil.</li>
            <li>Stir both mixtures together.</li>
            <li>Fill muffin tray 3/4 full.</li>
            <li>Bake for 20 minutes.</li>
        </ol>

Output -

  1. Mix flour, baking powder, sugar, and salt.
  2. In another bowl, mix eggs, milk, and oil.
  3. Stir both mixtures together.
  4. Fill muffin tray 3/4 full.
  5. Bake for 20 minutes.

Apart from global attributes, ordered list accepts three more attributes named reversed, start, type

Unordered List : <ul>

We use unorders list while don't need any order.

Below is an example.


        <ul>
            <li>first item</li>
            <li>second item</li>
            <li>third item</li>
        </ul>

Output -

  • first item
  • second item
  • third item

Apart from global attributes, unordered list accepts two more attributes named compact & type. But we will rather use css to alter the compact and type properties

Links

External resource link : <link>

We use this external resource link element to link with stylesheets and any other resources not present in the current webpage displayed. This external link can be linked with another folder in our app structure, or our hosting server or any other third party server. These don't render in webpage.

<link href="/media/examples/link-element-example.css" rel="stylesheet" />

Anchor element : <a>

The anhor element with its href (hyperlink reference), creates a hyperlink to web pages, files, email addresses, location in same page, or anywhere else a URL can address.

The href provides the location of the associated anchor.

Example
        <a href="https://www.google.com">Google Website

Results :

Google Website

Color of the content represents if the link is not visited, visited or active

Above is an example of absolute refrence. We can use relative reference in our server.

Example
        <a href="./index.html">Home Page

Results :

Home Page

        <li>Download a <a href="link-of-the-file" download>file</a></li>

Send email

Example: we need to provide a email address in 'href' attribute.
        <a href="mailto:ahsan@ahsanulkabir.dev">Send email to Ahsan</a>

Call

Sectioning Content

Sectioning contents used everywhere flow content is expected. Sectioning elements create a section that defines the scope of <header> element, <footer> elements and heading contents (h1 --- h6 & hgroup)

Elements like <article>, <aside>, <nav> and <section> falls under sectioning category.

Article : <article>

Articles are self-contained composition in a document, page, application, or site, which is intended to be independently distributatble. Examples - a forum post, a magazine or newspaper article, a blog entry, a product card, a user-submitted comment, an interactive widget, or any other independent content.

Generic Section Element : <section>

Section element does not have a more specific semantic element to represent it. It is used to represent generic standalone section of a document. Sections should always have a heading (few exceptions are there).

Navigation section element : <nav>

The <nav> HTML element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.

Example:
      <nav>
        <ul>
          <li><a href="#">Home</a></li>
          <li><a href="#">About</a></li>
          <li><a href="#">Contact</a></li>
        </ul>
      </nav>

Result:

  • Home
  • About
  • Contact

Image element : <img>

The <img> element embdeds an image into the document.

Example:
      <img src="./assets/images/orange.jpg" alt="Picture of a sliced orange">

Results:

Basic attributes of <img> elements

  • src - it is reequired. It contains the path of the image.
  • alt - this attribute holds a text descrilpt of the image which serves by rendering when the image cannot be loaded. Again, it help screen readers read the description of image.

Other attributes

  • width and height attributes used to set the size of the image and allowing to take up space and mitigate content layout shifts.
  • loading attribute determines how the browser will load the image. It has two values as below.
    • eager - immediately loads the image
    • lazy - Defers the loading the image until it reaches a calculated distance from viewport and hence avoid the network and storage of bandwidth requirement to handle the image until it is reasonable to download. By this way, it helps performance.

Figure and optional caption : <figure>, <figcaption>

The figure element represents a self-contained content, often with an optional caption which is used as <figcaption> element as a nested element within the tags of the <figure> element.

Example:
      <figure>
        <img width='660' height='480' src="https://interactive-examples.mdn.mozilla.net/media/cc0-images/elephant-660-480.jpg" alt="Elephant at sunset" />
        <figcaption>An elephant at sunset</figcaption>
      </figure>

Results:

Figure and figcaption not necessarily associated to img only. Those can be used with other contents.

Details element : <details>

This element creates a disclosure widget in which information is visition only when the widget is toggled into and "open" state. A summary or label must be provided using the <summary> element.

Example:
      <details>
        <summary>Details</summary>
        Something small enough to escape casual notice.
      </details>

Results

Details Something small enough to escape casual notice.

Note that, opening the summary element will take vertical space in the document rendered to show the details content.

Mark the highlight: <mark>

This elements used to highlight certain text. It is an inline-level element

Example:
      <details>
        <summary>Details of <mark>size</mark>/summary>
        Something small enough to escape casual notice.
      </details>

Results

Details of size Something small enough to escape casual notice.
